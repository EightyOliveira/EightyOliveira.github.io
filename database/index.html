MySQL--对varchar字段用int数值来查询的问题

1 示例 age是varchar类型，在下面两条sql中，第二条如果不加单引号，会查出其他很多无关数据 select * from table_name where age = '123' select * from table_name where age = 123 2 原因 官网原文 MySQL :: MySQL 5.7 Reference Manual :: 12.3 Type Conversion in Expression Evaluation 翻译与解释 在字符串和数值作比较的时候，MySQL会将字符串转换为DOUBLE类型（浮点数）进行比较。 字符串在转浮点数时，如果字符串开头是数字，那就一直截取，直到遇到非数字字符为止。若开头不是数字，转换结果为0。例如：‘123abc’ 转成 123，‘ab123’ 转成 0。 当varchar字段存储的是长数字字符串（如18位）时，问题尤为突出： 精度丢失：大整数转换为浮点数时，会使用科学计数法表示，且浮点数精度有限，无法精确表示超长整数，导致末尾数字被舍入或截断。 错误匹配：查询条件中的大整数和数据库中存储的长字符串，在转换为浮点数后可能变成相同的近似值，从而匹配到本不该匹配的记录。 索引失效：当使用整数查询varchar字段时，MySQL需要对索引字段的值进行隐式类型转换（相当于对字段使用了函数），这会导致该字段上的索引无法被使用，查询性能下降。 因此，对于varchar类型的字段，在查询时应始终使用字符串（即用引号括起来）进行匹配，避免使用整数数值，以保证查询的准确性和性能。 对比：Oracle Oracle数据库在处理此类比较时的行为与MySQL不同，通常不会出现因隐式转换导致的精度丢失和错误匹配问题。
