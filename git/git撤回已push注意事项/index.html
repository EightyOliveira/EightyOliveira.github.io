Git 撤回已 push 的注意事项

July 4, 2025
  • git revert 适合提交已经被推送到远程分支，适合公共分支，需要保留项目历史记录以便追踪问题。
  • 如果其他人已经拉取了你要撤销的提交，请优先使用 git revert。
  • git reset 会重写提交历史，特别是在使用 --hard 或 --force 参数时。如果你执行了 git push --force，可能会覆盖远程分支的历史，影响其他协作者。不适合用于已经推送到远程仓库并被他人拉取的提交。该命令适用于提交仅存在于本地，尚未推送到远程仓库的情况。
  • 在进行本地调试或清理提交历史时（如 rebase 后）可以使用。
  • 在执行任何撤销操作前，建议先创建一个新分支或备份当前状态。
  • 创建临时分支进行测试。
