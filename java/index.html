Arrays.asList()坑

基本用法 Arrays.asList() 是 Java 中一个常用的工具方法，用于将数组快速转换为 List。虽然使用方便，但如果不了解其特性，很容易踩坑。 String[] array = {"a", "b", "c"}; List<String> list = Arrays.asList(array); 注意事项（常见坑点） 1. 返回的 List 不是 java.util.ArrayList Arrays.asList() 返回的是 Arrays 类内部的一个私有静态类 java.util.Arrays.ArrayList，不是 java.util.ArrayList。 这个列表是固定大小的，不能添加或删除元素。 List<String> list = Arrays.asList("a", "b", "c"); list.add("d"); // 抛出 UnsupportedOperationException 解决办法：如果你需要一个可变长度的列表，应使用： List<String> list = new ArrayList<>(Arrays.asList("a", "b", "c")); 2. 原始数组与列表共享存储空间 对数组或返回的列表进行修改，都会影响对方，因为它们指向同一块内存。 String[] array = {"a", "b", "c"}; List<String> list = Arrays.asList(array); list.set(0, "z"); System.out.println(array[0]); // 输出: z 建议：如需独立副本，请手动复制： List<String> list = new ArrayList<>(Arrays.asList(array)); 3. 传入基本类型数组时会有“意外”结果 当你使用基本类型数组（如 int[]、boolean[]）时，会得到一个包含整个数组的单元素列表，而不是每个元素作为一个列表项。 int[] arr = {1, 2, 3}; List<int[]> list = Arrays.asList(arr); // 只有一个元素！ System.out.println(list.size()); // 输出: 1 原因：asList 的泛型不支持基本类型，数组会被当作一个对象处理。 ...

string的split小坑

示例说明 示例： String str = "a,b,c,,"; String[] result = str.split(","); System.out.println(Arrays.toString(result)); 输出： [a, b, c] 你可能会预期结果是 [a, b, c, "", ""]，但实际只有 [a, b, c]。 原因： 因为最后两个逗号之间是空字符串（""），Java 的 split() 方法会把这些尾部的空字符串当作“无意义”的内容自动去掉了。 技术细节 Java 中 String.split() 实际调用了 Pattern.split()，其默认行为如下： public String[] split(CharSequence input, int limit) 当你调用 split(regex) 时，底层其实是调用了 split(regex, 0) 如果 limit == 0，那么尾部的空字符串将不会保留在结果数组中 如何保留尾部的空字符串？ 如果你想保留这些尾部的空字符串，可以传入一个负数作为 limit 参数： String str = "a,b,c,,"; String[] result = str.split(",", -1); System.out.println(Arrays.toString(result)); 输出： [a, b, c, "", ""] 这样就可以保留所有的分割结果了。
