泛型擦除异常

1 问题描述 修复bug过程中，看到如下写法： public static void main(String[] args) { Map<String, Object> map = new HashMap<String, Object>(); map.put("1", 1); String result = (String) map.get("1"); } 异常信息： Exception in thread "main" java.lang.ClassCastException: java.lang.Integer cannot be cast to java.lang.String 2 原因分析 存入的是 Integer 类型：map.put("1", 1) 中的 1 是 int 基本类型，自动装箱为 Integer 对象存入 Map。 取出后强制转换：map.get("1") 返回的是 Integer 类型的对象，将其强制转换为 String 会导致类型转换异常。 泛型擦除的澄清：虽然 Java 泛型在运行时会被擦除（Map<String, Object> 变成原始类型 Map），但本例的问题并非由泛型擦除直接导致。真正原因是对象的实际类型与强制转换的目标类型不匹配。 3 解决方案 方案一：使用正确的接收类型 Integer result = (Integer) map.get("1"); 方案二：存入时即为 String 类型 map.put("1", "1"); // 存入字符串 "1" String result = (String) map.get("1");
